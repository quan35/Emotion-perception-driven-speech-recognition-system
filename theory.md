# 情感识别模型理论详解

本文档详细阐述了项目中使用的两种核心情感识别模型：
1.  **自建模型**: 一个从零开始构建的、融合了多种现代深度学习技术的 `CNN+BiLSTM+Attention` 模型。
2.  **共享模型**: 基于 OpenAI `Whisper` 预训练模型的强大特征提取能力的迁移学习模型。

---

# A. 自建模型: EmotionRecognizer

## 1. 设计思路

语音情感识别（Speech Emotion Recognition, SER）的核心挑战在于：情感信息同时蕴含在语音的**局部声学特征**（如音高突变、能量爆发）和**全局时序模式**（如语速变化趋势、语调走向）中。单一模型难以同时捕获这两个层次的信息。

本模型采用 **CNN + BiLSTM + Attention** 三阶段级联架构：

```
Mel 频谱图 → 残差SE-CNN(局部特征) → BiLSTM(时序建模) → 多头注意力(关键帧聚合) → 分类头 → 情感类别
```

每个模块承担明确的职责：

| 阶段 | 模块 | 职责 |
|---|---|---|
| 1 | **残差 SE-CNN** | 从频谱图中高效提取鲁棒的局部声学模式。 |
| 2 | **BiLSTM** | 沿时间轴双向建模局部特征间的上下文依赖关系。 |
| 3 | **多头注意力** | 从多个子空间自适应聚焦并加权聚合情感最强烈的时序特征。 |
| 4 | **分类头** | 将聚合后的高维特征映射到 6 种情感类别。 |


## 2. 输入表示：Mel 频谱图

原始音频波形是一维时序信号，直接建模效率低。本系统先将音频转换为 **Mel 频谱图**——一种二维时频表示：

- **横轴**：时间帧（每帧约 32ms）
- **纵轴**：Mel 频率通道（`n_mels=128`）
- **值**：该时间-频率位置的对数能量

对于 5 秒、16kHz 的音频，Mel 频谱图的尺寸约为 **(128, 157)**。以单通道图像形式送入模型，张量形状为 `(batch, 1, 128, 157)`。

**选择 Mel 频谱图的原因**：Mel 刻度模拟人耳对频率的非线性感知，低频分辨率高、高频分辨率低，与人类对语音情感的感知特性一致。

## 3. 第一阶段：残差 SE-CNN 局部特征提取

该阶段由 3 个级联的 `_ResidualCNNBlock` 组成，相比传统 CNN，它融合了残差连接（Residual Connection）和通道注意力（Squeeze-and-Excitation）机制，旨在解决深度网络训练中的梯度消失问题，并让模型关注信息量最大的特征通道。

### 3.1 核心组件

- **残差连接 (Residual Connection)**: 允许梯度通过“快捷通道”直接回传，缓解深度网络中的梯度消失问题，使得更深层的网络更容易训练。
- **SE 通道注意力 (_SEBlock)**: 通过“Squeeze”（全局平均池化）和“Excitation”（两层全连接网络）操作，动态学习每个特征通道的重要性，然后对通道进行加权，放大关键特征（如共振峰、音高相关特征），抑制无关特征。
- **Spatial Dropout (Dropout2d)**: 在特征图层面随机丢弃整个通道，强迫网络学习更多样化的特征，是 CNN 中一种有效的正则化手段。

### 3.2 优化后的完整流程

```
输入 Mel 频谱图 (batch, 1, 128, 157)
    ↓ [训练时] 高斯噪声注入 (std=0.01)
    ↓ 残差 SE-CNN 第1层 (Conv+BN+ReLU → SE → 残差 → Pool → Dropout2d)
(batch, 32, 64, 78)
    ↓ 残差 SE-CNN 第2层
(batch, 64, 32, 39)
    ↓ 残差 SE-CNN 第3层
(batch, 128, 16, 19)
```

## 4. 第二阶段：BiLSTM 时序建模

CNN 的输出被重塑为 `(batch, time=19, features=2048)` 的序列，送入一个双层双向 LSTM (`BiLSTM`)。

- **为什么用 BiLSTM**: 情感在语音中是**上下文相关**的。一个词的情感可能依赖于前后的词语和语调。BiLSTM 通过前向和后向两个 LSTM 网络，使每个时间步的输出都能融合过去和未来的信息，从而获得更全面的上下文表示。

- **输出**: BiLSTM 输出一个 `(batch, 19, 128)` 的张量，其中每个时间步的 128 维向量都编码了该时刻的声学特征及其在整个语音中的时序上下文。

## 5. 第三阶段：多头注意力加权聚合

BiLSTM 输出了一个特征序列，但分类器需要一个固定长度的向量。简单的平均或取最后一步会丢失信息。**多头注意力（Multi-Head Attention）** 机制被用来解决这个问题。

### 5.1 优化六：多头注意力 (Multi-Head Attention)

**问题**：原始单头注意力只能学习一种"关注模式"。但语音中的情感线索是多维的——节奏、音高、能量等。单头注意力被迫将所有维度的信息压缩为一组权重，表达能力有限。

**方案**：用 4 头缩放点积注意力（scaled dot-product attention）替代单头注意力。

**原理**：
- **多头并行**：模型将 128 维特征拆分为 4 个“头”，每个头在独立的 32 维子空间中学习不同的关注模式。例如，一个头可能关注音高变化，另一个头关注能量爆发。
- **信息聚合 (修正后)**:
    1.  **自注意力变换**: 在每个头内部，通过 `Q`, `K`, `V` 矩阵计算自注意力，生成一个上下文感知的序列 `context` (shape: `B, T, D`)，此时每个时间步都已融合了全局信息。
    2.  **时间维加权聚合**: 与简单的对时间维做平均池化不同，**修复后的版本采用了一种更有效的聚合策略**。它从多头注意力分数 `attn` 中派生出一组新的聚合权重，并用这组权重对 `context` 序列进行**加权求和**。
    3.  这种方法直接利用注意力机制的核心——即识别重要时间帧的能力——来聚合信息，避免了简单平均可能带来的信息稀释问题，从而得到一个更能代表整个序列情感焦点的上下文向量。

## 6. 第四阶段：分类头

最终，由注意力机制聚合的 `(batch, 128)` 上下文向量被送入一个简单的全连接分类头，输出 6 个情感类别的得分（logits）。

```python
Linear(128 → 64) → ReLU → Dropout(0.5) → Linear(64 → 6)
```

## 7. 其他优化策略

为提升模型性能和泛化能力，还采用了一系列先进的训练策略：
- **高斯噪声注入**: 在数据和模型层面引入噪声，作为一种正则化手段，提升模型对真实噪声环境的鲁棒性。
- **LayerNorm**: 在 BiLSTM 和 Attention 之间加入，稳定 Attention 层的输入，使其更关注内容而非数值量级。
- **权重初始化**: 对不同类型的层（Conv, LSTM, Linear）采用最适合的初始化方法（Kaiming, Orthogonal, Xavier），加速模型收敛。
- **AdamW 优化器**: 采用解耦权重衰减的 AdamW，正则化效果优于标准 Adam。
- **学习率预热与调度**: 采用线性 Warmup 和 ReduceLROnPlateau 策略，使训练过程更稳定。
- **Label Smoothing / Focal Loss**: 使用标签平滑来防止模型过自信，并提供 Focal Loss 选项以应对潜在的类别不平衡问题。

# B. 共享模型: WhisperEmotionHead

与自建模型从零训练不同，`WhisperEmotionHead` 模型采取了**迁移学习 (Transfer Learning)** 的策略，旨在利用大型预训练模型的强大能力来解决情感识别任务。

## 1. 设计思路：站在巨人的肩膀上

- **核心思想**: OpenAI 的 `Whisper` 模型在海量的、多样化的（超过68万小时）音频数据上进行了预训练，主要任务是自动语音识别（ASR）。在这个过程中，它的 **Encoder** 部分已经学会了如何从音频频谱图中提取极其丰富和鲁棒的通用声学特征。这些特征不仅对语音内容识别有用，对情感、语速、说话人身份等副语言信息同样具有强大的表征能力。
- **迁移策略**: 我们不必从零开始学习这些基础特征，而是可以直接“借用”Whisper Encoder 作为我们的特征提取器。我们冻结 Encoder 的大部分（或全部）参数，只训练一个附加在它上面的、小型的、任务相关的**分类头**。

```
Whisper Mel 频谱图 → [冻结的] Whisper Encoder (通用声学特征) → 平均池化(聚合) → 分类头 → 情感类别
```

## 2. 模型结构 (`WhisperEmotionHead`)

### 2.1 输入: Whisper Mel 频谱图

- Whisper 使用与自建模型不同的频谱图参数：`n_mels=80`，且音频被统一填充/裁剪为 30 秒，对应 `3000` 个时间帧。输入形状为 `(batch, 80, 3000)`。

### 2.2 特征提取: Whisper Encoder

- **架构**: Whisper Encoder 是一个 **Transformer** 网络。它由一系列的 Transformer Block 堆叠而成，每个 Block 内部都包含多头自注意力层和前馈网络。
- **功能**: 它接收频谱图序列，输出一个等长的特征序列，形状为 `(batch, 1500, D)`，其中 `D` 是模型的隐藏维度（如 `base` 模型为 512）。这个输出的特征序列是高度抽象的声学表征。
- **参数冻结**: 在 `WhisperEmotionHead` 中，默认**冻结**整个 Encoder 的参数 (`freeze_encoder=True`)。这意味着在我们的情感分类任务训练中，Encoder 的权重不会被更新。这有两个好处：
    1.  **防止灾难性遗忘**: 避免在小的情感数据集上破坏 Whisper 在大数据集上学到的通用知识。
    2.  **训练高效**: 可训练参数极少，大大减少了训练时间和显存需求。

### 2.3 特征聚合: 平均池化

- Whisper Encoder 输出的是一个序列 `(batch, time, features)`。为了得到一个用于分类的固定长度向量，模型采用 `AdaptiveAvgPool1d` 对时间维度进行**平均池化**。这是一种简单而有效的方法，可以获得整个语音片段的全局表示。

### 2.4 分类头

- 一个简单的多层感知机（MLP）：
  ```python
  Linear(D → 256) → ReLU → Dropout → Linear(256 → 64) → ReLU → Dropout → Linear(64 → 6)
  ```
- 这个分类头非常轻量，它的任务是从 Whisper 提取的通用特征中，学习如何解码出情感信息。

## 3. 变体：`WhisperEmotionFinetune`

这是一个备选方案，用于数据量相对更充足的场景。

- **策略**: 它不是完全冻结 Encoder，而是**解冻最后几层** (`unfreeze_last_n=2`)。
- **原理**: 模型的基础声学特征（由浅层Encoder提取）保持不变，而更抽象的高层语义特征（由深层Encoder提取）则有机会根据情感分类任务进行微调（Finetune），以更好地适配新任务。这通常能带来比完全冻结更高的性能，但需要更多的数据和计算资源。

# C. 模型对比：自建模型 vs. 共享模型

| 特性 | **自建模型 (EmotionRecognizer)** | **共享模型 (WhisperEmotionHead)** |
|---|---|---|
| **核心架构** | `残差SE-CNN + BiLSTM + 多头注意力` | `Transformer Encoder + 平均池化` |
| **训练范式** | **从零训练 (Training from Scratch)** | **迁移学习 (Transfer Learning)** |
| **知识来源** | 完全依赖当前提供的情感数据集。 | 依赖 Whisper 68万小时的预训练知识 + 当前情感数据。 |
| **数据需求** | 相对更高。需要足够数据从零学习有效的声学特征。 | 相对更低。预训练模型提供了强大的特征先验。 |
| **模型参数量** | 总计 **~1.4M**，全部参与训练。 | `base`模型总计 **~74M**，但只训练分类头（**~150K** 参数）。 |
| **特征特异性** | 学习到的特征**完全针对情感分类任务**。 | 特征是**通用声学特征**，分类头负责适配情感任务。 |
| **灵活性与可控性** | **高**。可以精细调整架构的每个组件（CNN, LSTM等）。 | **低**。主要依赖预训练的Encoder，可调整部分有限。 |
| **性能潜力** | 在特定数据集上可能达到很高性能，但泛化能力依赖数据质量和精心调优。 | 通常有很高的性能下限和上限，对不同领域音频的鲁棒性更强。 |
| **训练成本** | 中等。需要完整训练整个网络。 | **极低**。只需训练一个微小的分类头，速度飞快。 |
| **最佳适用场景** | 数据集相对干净、规模中等，需要高度可控和可解释的定制化模型。 | 数据量有限，或需要快速验证一个高性能基线模型。追求最佳泛化能力。 |
