# CNN+BiLSTM+Attention 情感识别模型 — 理论设计

## 1. 设计思路

语音情感识别（Speech Emotion Recognition, SER）的核心挑战在于：情感信息同时蕴含在语音的**局部声学特征**（如音高突变、能量爆发）和**全局时序模式**（如语速变化趋势、语调走向）中。单一模型难以同时捕获这两个层次的信息。

本模型采用 **CNN + BiLSTM + Attention** 三阶段级联架构：

```
Mel 频谱图 → CNN（局部特征） → BiLSTM（时序建模） → Attention（关键帧聚合） → 分类头 → 情感类别
```

每个模块承担明确的职责：

| 阶段 | 模块 | 职责 |
|---|---|---|
| 1 | CNN | 从频谱图中提取局部声学模式（类似图像中的纹理） |
| 2 | BiLSTM | 沿时间轴双向建模上下文依赖关系 |
| 3 | Attention | 自适应聚焦情感表达最强烈的时间帧 |
| 4 | 分类头 | 将聚合后的特征映射到 6 种情感类别 |

## 2. 输入表示：Mel 频谱图

原始音频波形是一维时序信号，直接建模效率低。本系统先将音频转换为 **Mel 频谱图**——一种二维时频表示：

- **横轴**：时间帧（每帧对应 hop_length=512 个采样点，约 32ms）
- **纵轴**：Mel 频率通道（n_mels=128）
- **值**：该时间-频率位置的对数能量

对于 5 秒、16kHz 的音频，Mel 频谱图的尺寸约为 **(128, 157)**，即 128 个频率通道、157 个时间帧。

以单通道图像形式送入模型，张量形状为 `(batch, 1, 128, 157)`。

**选择 Mel 频谱图的原因**：Mel 刻度模拟人耳对频率的非线性感知，低频分辨率高、高频分辨率低，与人类对语音情感的感知特性一致。

## 3. 第一阶段：CNN 局部特征提取

### 3.1 结构

三层 CNN 模块，每层包含：

```
Conv2d(3×3, padding=1) → BatchNorm2d → ReLU → MaxPool2d(2×2)
```

通道数逐层递增：**1 → 32 → 64 → 128**。

### 3.2 维度变化

以输入 `(batch, 1, 128, 157)` 为例：

| 层 | 操作 | 输出形状 |
|---|---|---|
| 输入 | — | (batch, 1, 128, 157) |
| 第 1 层 | Conv2d(1→32) + BN + ReLU + Pool | (batch, 32, 64, 78) |
| 第 2 层 | Conv2d(32→64) + BN + ReLU + Pool | (batch, 64, 32, 39) |
| 第 3 层 | Conv2d(64→128) + BN + ReLU + Pool | (batch, 128, 16, 19) |

每层的 MaxPool2d(2) 将频率和时间维度各缩小一半。三层后：
- 频率维度：128 → 64 → 32 → **16**（即 n_mels ÷ 8）
- 时间维度：157 → 78 → 39 → **19**

### 3.3 设计原理

- **Conv2d(3×3, padding=1)**：3×3 卷积核在频谱图上滑动，捕获局部时频模式。padding=1 保持特征图尺寸不变（缩减由 MaxPool 完成）。这些局部模式包括：特定频段的能量分布、共振峰结构、谐波模式等。
- **BatchNorm2d**：对每个通道做归一化，加速收敛、缓解内部协变量偏移，允许使用更大的学习率。
- **ReLU**：引入非线性，使网络能学习复杂的特征映射。使用 inplace=True 节省显存。
- **MaxPool2d(2×2)**：下采样，扩大后续卷积层的感受野。同时具有一定的平移不变性——即使同一情感特征在频谱图上有微小位移，池化后仍能被捕获。
- **通道数递增（32→64→128）**：浅层用少量通道捕获简单模式（如边缘、能量带），深层用更多通道组合出复杂的高级特征。

## 4. CNN 到 BiLSTM 的维度转换

CNN 输出 `(batch, 128, 16, 19)` 后，需要重新组织为时间序列形式才能送入 LSTM。

```python
# (batch, channels=128, freq=16, time=19)
out = out.permute(0, 3, 1, 2)    # → (batch, time=19, channels=128, freq=16)
out = out.view(batch, time, -1)   # → (batch, time=19, 128×16=2048)
```

**含义**：将每个时间帧上的所有通道和频率信息拼接成一个 2048 维的特征向量。这样就得到了一个长度为 19、每步特征维度为 2048 的时间序列。

## 5. 第二阶段：BiLSTM 时序建模

### 5.1 结构

```
BiLSTM(input_size=2048, hidden_size=64, num_layers=2, bidirectional=True)
```

输出维度 = 64 × 2 = **128**（前向 64 维 + 反向 64 维）。

### 5.2 为什么用 BiLSTM

情感在语音中是**上下文相关**的。例如：
- 一句话开头的低沉语调，可能在结尾突然升高——需要结合后文才能判断是"悲伤"还是"惊讶"
- 语速的加快只有与前后对比才有意义

**双向 LSTM** 同时从两个方向扫描序列：
- **前向**：从 t=0 到 t=18，捕获"到目前为止发生了什么"
- **反向**：从 t=18 到 t=0，捕获"之后将发生什么"

每个时间步的输出融合了前后文信息，比单向 LSTM 更全面。

### 5.3 多层堆叠

使用 2 层 LSTM 堆叠（num_layers=2），第一层的输出作为第二层的输入。层间使用 dropout=0.3 防止过拟合。多层堆叠使网络能学习更抽象的时序特征。

### 5.4 输出

BiLSTM 输出 `(batch, 19, 128)`，即 19 个时间步，每步 128 维特征向量。每个向量都编码了该时刻的局部声学特征及其在整个语音片段中的上下文信息。

## 6. 第三阶段：Attention 加权聚合

### 6.1 动机

BiLSTM 输出了 19 个时间步的特征，但最终分类需要一个固定维度的向量。简单的方法是取平均或取最后一步，但这忽略了一个事实：**不是每个时间帧都同等重要**。

在情感语音中，情感信息往往集中在某些关键时刻——如重读音节、语调转折处。Attention 机制让模型**自动学习关注哪些时间帧**。

### 6.2 计算过程

```python
# h_t: BiLSTM 在时间步 t 的输出, 维度 128
e_t = Tanh(W_1 · h_t + b_1)     # 线性变换 + 激活, 128 → 64
s_t = W_2 · e_t                  # 打分, 64 → 1 (无偏置)
α_t = softmax(s_t)               # 归一化为概率权重, 沿时间维
context = Σ (α_t · h_t)          # 加权求和, 得到 128 维上下文向量
```

用数学公式表示：

```
e_t = tanh(W₁ h_t + b₁)
α_t = exp(w₂ᵀ e_t) / Σⱼ exp(w₂ᵀ eⱼ)
c = Σ_t α_t · h_t
```

### 6.3 直觉理解

- **W₁ (128→64) + Tanh**：将每个时间步的特征投影到注意力空间，Tanh 将值压缩到 [-1, 1]
- **W₂ (64→1)**：为每个时间步打一个标量分数，分数越高表示该帧越重要
- **softmax**：将分数归一化为概率分布，所有时间步的权重之和为 1
- **加权求和**：重要帧的特征被放大，不重要的被抑制

最终得到一个 **128 维的上下文向量** `context`，浓缩了整段语音中最具情感辨别力的信息。

### 6.4 可解释性

Attention 权重 α_t 可以可视化：权重高的时间帧对应语音中情感表达最强烈的片段，这为模型的决策提供了可解释性。

## 7. 第四阶段：分类头

```python
Linear(128 → 64) → ReLU → Dropout(0.5) → Linear(64 → 6)
```

- **Linear(128→64)**：降维，将上下文向量压缩到更紧凑的表示
- **ReLU**：非线性激活
- **Dropout(0.5)**：训练时随机丢弃 50% 的神经元。分类头参数量较大，容易过拟合，因此使用较高的 dropout 率
- **Linear(64→6)**：输出 6 个情感类别的原始分数（logits）

### 推理时的概率输出

```python
probs = softmax(logits)   # 将 logits 转为概率分布
predicted = argmax(probs)  # 概率最大的类别即为预测结果
confidence = max(probs)    # 最大概率值作为置信度
```

## 8. 完整数据流

以一条 5 秒音频为例，batch_size=1：

```
输入音频 (5s, 16kHz)
    ↓ Mel 频谱图提取
(1, 1, 128, 157)           — 单通道"图像"
    ↓ CNN 第1层
(1, 32, 64, 78)
    ↓ CNN 第2层
(1, 64, 32, 39)
    ↓ CNN 第3层
(1, 128, 16, 19)           — 128个特征图, 16频率×19时间
    ↓ 维度重排
(1, 19, 2048)              — 19个时间步, 每步2048维
    ↓ BiLSTM (2层, 双向)
(1, 19, 128)               — 19个时间步, 每步128维(64前向+64反向)
    ↓ Attention 加权聚合
(1, 128)                   — 单个上下文向量
    ↓ 分类头
(1, 6)                     — 6个情感类别的 logits
    ↓ softmax
(1, 6)                     — 概率分布 [happy, angry, sad, neutral, fear, surprise]
```

## 9. 模型参数量估算

| 模块 | 参数量 |
|---|---|
| CNN 第1层 Conv(1→32) + BN | 32×1×3×3 + 32×2 = 352 |
| CNN 第2层 Conv(32→64) + BN | 64×32×3×3 + 64×2 = 18,560 |
| CNN 第3层 Conv(64→128) + BN | 128×64×3×3 + 128×2 = 73,984 |
| BiLSTM 第1层 | 4×(2048+64+1)×64 × 2 = 1,082,112 |
| BiLSTM 第2层 | 4×(128+64+1)×64 × 2 = 98,816 |
| Attention (128→64→1) | 128×64+64 + 64×1 = 8,320 |
| 分类头 (128→64→6) | 128×64+64 + 64×6+6 = 8,646 |
| **合计** | **约 1.29M** |

模型参数量约 **129 万**，属于轻量级网络，在单卡 GPU 上训练速度快、显存占用小。

## 10. 训练策略

| 项目 | 配置 | 说明 |
|---|---|---|
| 损失函数 | CrossEntropyLoss | 多分类标准损失，内含 softmax |
| 优化器 | Adam (lr=0.001, weight_decay=1e-4) | 自适应学习率，L2 正则化防止过拟合 |
| 学习率调度 | ReduceLROnPlateau (patience=5, factor=0.5) | 验证损失 5 轮不下降则学习率减半 |
| 早停 | patience=10 | 验证损失 10 轮不下降则停止训练 |
| 数据划分 | 训练:验证:测试 = 8:1:1 | 随机划分，固定种子 seed=42 保证可复现 |
| 最大训练轮数 | 100 epochs | 通常在 30-50 轮内收敛 |

## 11. 设计选择的理由总结

**为什么不直接用 CNN 分类？** CNN 擅长提取局部特征，但缺乏对长程时序依赖的建模能力。情感的表达常跨越多个时间帧，需要 LSTM 来捕获。

**为什么不直接用 LSTM？** 原始 Mel 频谱图每帧 128 维，信息冗余大。CNN 先做降维和特征抽象，再送入 LSTM，既减少了 LSTM 的计算负担，也让 LSTM 能在更有意义的特征空间上建模。

**为什么用 Attention 而不是取 LSTM 最后一步？** LSTM 最后一步的隐状态会偏向序列末端的信息。Attention 让模型自动学习关注任意位置的关键帧，不受位置偏差影响，且提供了可解释性。

**为什么用 BiLSTM 而不是单向 LSTM？** 情感判断需要全局上下文。例如一段语音前半段平淡、后半段激动，单向 LSTM 在前半段时无法利用后半段信息，双向 LSTM 则可以。

## 12. 优化设计

基于实践经验和相关研究，对原始模型进行了四项针对性优化。优化后的完整数据流为：

```
输入 Mel 频谱图 (batch, 1, 128, 157)
    ↓ [训练时] 高斯噪声注入 (std=0.01)
    ↓ 残差 CNN 第1层 (Conv+BN+ReLU + 残差 + Pool + Dropout2d)
(batch, 32, 64, 78)
    ↓ 残差 CNN 第2层
(batch, 64, 32, 39)
    ↓ 残差 CNN 第3层
(batch, 128, 16, 19)
    ↓ 维度重排
(batch, 19, 2048)
    ↓ BiLSTM (2层, 双向)
(batch, 19, 128)
    ↓ LayerNorm
(batch, 19, 128)
    ↓ Attention 加权聚合
(batch, 128)
    ↓ 分类头
(batch, 6)
```

### 12.1 优化一：CNN 残差连接 + Spatial Dropout

**问题**：原始模型的三层 CNN 是纯顺序堆叠。随着层数加深，梯度信号在反向传播时逐层衰减，浅层参数更新缓慢。

**方案**：将每个 CNN 层改造为残差块（Residual Block），结构为：

```
       ┌──────────────────────────────────────┐
       │                                      │
输入 x ─→ Conv2d(3×3) → BN → ReLU ─→ (+) ─→ MaxPool ─→ Dropout2d ─→ 输出
       │                              ↑      │
       └── shortcut (1×1 Conv + BN) ──┘      │
           (仅当 in_ch ≠ out_ch 时)           │
       └──────────────────────────────────────┘
```

**原理**：
- **残差连接（Skip Connection）**：让梯度可以绕过卷积层直接回传，缓解梯度消失。即使卷积层学到的是零映射，输出仍等于输入，保证"至少不比浅层差"。当输入输出通道数不同时（如 1→32），用 1×1 卷积做通道对齐。
- **Spatial Dropout（Dropout2d, p=0.1）**：与普通 Dropout 随机丢弃单个元素不同，Dropout2d 随机丢弃整个特征通道。这对 CNN 更有效，因为同一通道内的激活值高度相关，丢弃整个通道能打破这种相关性，迫使网络不依赖单一特征。选择较小的丢弃率 0.1 是因为 CNN 部分参数量不大，过高会损害特征提取能力。
- **残差加法在 MaxPool 之前**：先做残差融合再池化，保证主路径和残差路径在相同的空间分辨率上对齐。

### 12.2 优化二：训练时高斯噪声注入

**问题**：RAVDESS 等数据集在录音棚中采集，非常干净。模型容易"死记硬背"这些干净信号的噪声模式，导致在真实环境（有背景噪声、混响）中表现下降。

**方案**：在模型 `forward()` 的第一步，当处于训练模式（`self.training=True`）时，对输入 Mel 频谱图叠加微小高斯噪声：

```
x_noisy = x + ε,    ε ~ N(0, σ²),    σ = 0.01
```

**原理**：
- 高斯噪声注入本质上是一种**正则化手段**，等价于在损失函数中加入了关于输入的梯度惩罚。这鼓励模型学习对输入微小扰动不敏感的决策边界，提升泛化能力。
- σ=0.01 是一个经验值。Mel 频谱图经过对数缩放后值域通常在 [-80, 0] dB 量级（归一化后约 [0, 1]），0.01 的噪声相当于约 1% 的扰动，不会破坏情感信息但足以打乱过拟合的记忆模式。
- 仅在训练时生效（`model.eval()` 时自动关闭），不影响推理性能。
- 实践表明，适度的噪声注入可以提升验证集准确率 2-4 个百分点。

**双重噪声策略**：本系统在两个层面都引入了噪声注入：
1. **数据增强层面**（`AudioAugmentation`）：在频谱图 numpy 数组上加噪，发生在数据加载阶段
2. **模型层面**（`forward()` 内）：在 PyTorch 张量上加噪，发生在前向传播阶段

两者互补——数据增强的噪声对每个样本是固定的（加载时确定），模型内部的噪声每次前向传播都不同，增加了随机性。

### 12.3 优化三：BiLSTM 后加 LayerNorm

**问题**：BiLSTM 的输出数值范围不稳定，不同时间步、不同样本的输出量级可能差异很大。这导致 Attention 层的权重计算受到数值尺度的干扰——本应关注"内容"，却被"量级"误导。

**方案**：在 BiLSTM 输出和 Attention 之间插入 LayerNorm：

```
BiLSTM 输出 (batch, time, 128) → LayerNorm(128) → Attention
```

**原理**：
- **LayerNorm** 对每个样本的每个时间步独立地进行归一化：将 128 维向量的均值调整为 0、标准差调整为 1，然后通过可学习的缩放和偏移参数恢复表达能力。
- 归一化后，不同时间步的特征处于统一的数值尺度上，Attention 的打分函数可以更公平地比较各时间步的"重要性"，而非被数值量级干扰。
- 与 BatchNorm 不同，LayerNorm 不依赖 batch 维度的统计量，因此在 batch_size 较小或推理时表现稳定。

### 12.4 优化四：增强数据增强

**问题**：原有的 `AudioAugmentation` 只包含 SpecAugment（时间遮蔽 + 频率遮蔽），增强手段单一。

**方案**：新增频谱图级别的高斯噪声注入，使数据增强同时包含：

| 增强方法 | 作用 | 参数 |
|---|---|---|
| 时间遮蔽 | 随机遮盖连续时间帧，模拟语音片段缺失 | max_width=20 |
| 频率遮蔽 | 随机遮盖连续频率通道，模拟频段缺失 | max_width=10 |
| **高斯噪声** | 全局添加随机扰动，模拟环境噪声 | **std=0.01** |

三者配合使用，分别从"信息缺失"和"信息扰动"两个维度增加训练数据的多样性。

### 12.5 优化后的参数量变化

| 新增模块 | 新增参数量 |
|---|---|
| 残差 shortcut 1×1 Conv (1→32) + BN | 32 + 64 = 96 |
| 残差 shortcut 1×1 Conv (32→64) + BN | 2,048 + 128 = 2,176 |
| 残差 shortcut 1×1 Conv (64→128) + BN | 8,192 + 256 = 8,448 |
| LayerNorm(128) | 256 |
| **新增合计** | **约 11K** |

优化后总参数约 **1.30M**（增加不到 1%），模型仍属轻量级，训练速度几乎不受影响。

### 12.6 接口兼容性

新增的构造参数 `cnn_dropout=0.1` 和 `noise_std=0.01` 均有默认值，所有调用方（`inference/pipeline.py`、`notebooks/03_train_emotion.ipynb`）无需修改即可使用优化后的模型。`forward()` 的输入输出形状完全不变。

**注意**：由于模型结构发生了变化（新增残差 shortcut 层和 LayerNorm），旧的权重文件 `best_emotion.pth` 无法直接加载到优化后的模型中，需要重新训练。
